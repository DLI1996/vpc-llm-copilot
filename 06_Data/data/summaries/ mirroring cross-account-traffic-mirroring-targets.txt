This documentation provides information on how to share and manage traffic mirror targets in Amazon VPC using AWS Resource Access Manager (RAM). It explains how to share a traffic mirror target with other AWS accounts, accept a resource share from another account, and delete a resource share. It also mentions that cross-account traffic mirror targets are possible, where the target can be owned by a different AWS account than the traffic mirror source. The documentation provides step-by-step instructions for sharing, accepting, and deleting resource shares using the AWS RAM console.
