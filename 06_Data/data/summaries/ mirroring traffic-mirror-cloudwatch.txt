This document provides information on how to monitor and troubleshoot Traffic Mirroring in Amazon VPC using Amazon CloudWatch. It lists the available CloudWatch metrics for mirrored traffic, such as the number of bytes received and sent, and the number of packets received and sent. It also explains the dimensions that can be used to filter the metric data, such as AutoScalingGroupName, ImageId, InstanceId, and InstanceType. The document provides instructions on how to view the Traffic Mirroring metrics using the CloudWatch console or the AWS CLI.
