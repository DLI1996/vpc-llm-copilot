This documentation is about the AWS Global Networks for Transit Gateways API, specifically the DeregisterTransitGateway action. This action allows you to deregister a transit gateway from your global network, removing any customer gateway associations. The request requires the global network ID and the transit gateway ARN as parameters. The response includes the transit gateway registration information. Possible errors include access denied, conflict, internal server error, resource not found, throttling, and validation failure.
