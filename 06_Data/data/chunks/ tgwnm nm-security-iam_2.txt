With IAM identity-based policies, you can specify allowed or denied actions and resources, and specify the conditions under which actions are allowed or denied. Network Manager supports specific actions, resources, and condition keys. For a complete list, see Actions, Resources, and Condition Keys for Network Manager in the IAM User Guide.To learn about all of the elements that you use in a JSON policy, see IAM JSON Policy Elements Reference in the IAM User Guide.ActionsPolicy actions in Network Manager use the following prefix before the action: networkmanager:. For example, to grant someone permission to create a global network with the CreateGlobalNetwork API operation, you include the networkmanager:CreateGlobalNetwork action in their policy.For a list of global networks actions, see the Network Manager API Reference.Resources
