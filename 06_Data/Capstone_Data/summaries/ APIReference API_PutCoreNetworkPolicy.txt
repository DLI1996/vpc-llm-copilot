This documentation is about the AWS Global Networks for Transit Gateways API, specifically the PutCoreNetworkPolicy operation. This operation creates a new version of a core network policy and shows the differences between the live policy and the submitted policy. The request syntax includes parameters such as client token, description, latest version ID, and policy document. The response syntax includes elements such as core network policy alias, change set state, core network ID, creation timestamp, description, policy document, and policy errors. The documentation also provides information about possible errors and related API references.
