This document provides information about sharing a transit gateway in AWS. It explains that AWS Resource Access Manager (RAM) can be used to share a transit gateway across accounts or within an organization. It also mentions that certain considerations need to be taken into account when sharing a transit gateway, such as creating an AWS Site-to-Site VPN attachment in the same account as the transit gateway. The document also discusses permissions and operations that can be performed by the resource owner and the shared account. It concludes by mentioning that when a transit gateway is unshared, the transit gateway attachment remains functional and the transit gateway itself is not impacted. Additionally, it briefly mentions shared subnets and provides links to further documentation.
