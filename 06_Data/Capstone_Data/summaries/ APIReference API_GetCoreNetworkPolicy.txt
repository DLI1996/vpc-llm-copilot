This documentation is about the AWS Global Networks for Transit Gateways API, specifically the GetCoreNetworkPolicy operation. It provides details about a core network policy, including the alias, change set state, core network ID, creation date, description, policy document, policy errors, and policy version ID. The request parameters include the alias, core network ID, and policy version ID. The response is in JSON format and includes the core network policy details. Possible errors include access denied, internal server error, resource not found, throttling, and validation failure. The documentation also provides links to language-specific AWS SDKs for using the API.
