This documentation provides information on how to control the use of VPC endpoints in AWS PrivateLink. It explains how to control the creation of VPC endpoints based on the service owner, control the private DNS names and service names that can be specified for VPC endpoint services. It also provides identity-based policy examples for AWS PrivateLink. By default, users do not have permission to create or modify AWS PrivateLink resources, but an IAM administrator can create IAM policies to grant users the necessary permissions. The documentation includes JSON policy examples for controlling the use of VPC endpoints, controlling VPC endpoints creation based on the service owner, controlling the private DNS names, and controlling the service names.
