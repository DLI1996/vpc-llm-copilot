The first four IPv6 addresses and the last IPv6 address in each subnet CIDR block are not available for your use, and they cannot be assigned to an EC2 instance. For example, in a subnet with CIDR block 2001:db8:1234:1a00/64, the following five IP addresses are reserved:2001:db8:1234:1a00::2001:db8:1234:1a00::1: Reserved by AWS for the VPC router.2001:db8:1234:1a00::22001:db8:1234:1a00::32001:db8:1234:1a00:ffff:ffff:ffff:ffffIn addition to the IP address reserved by AWS for the VPC router in the example above, the following IPv6 addresses are reserved for the default VPC router:A link-local IPv6 address in the FE80::/10 range generated using EUI-64. For more information about link-local addresses, see Link-local address.The link-local IPv6 address FE80:ec2::1.If you need to communicate with the VPC router over IPv6, you can configure your applications to communicate with whichever address best fits your need.
