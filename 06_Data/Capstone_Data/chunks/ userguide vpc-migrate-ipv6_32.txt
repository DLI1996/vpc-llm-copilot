RHEL 7.4 and CentOS 7 and later use cloud-init to configure your network interface and generate the /etc/sysconfigbetwork-scripts/ifcfg-eth0 file. You can create a custom cloud-init configuration file to enable DHCPv6, which generates an ifcfg-eth0 file with settings that enable DHCPv6 after each reboot.NoteDue to a known issue, if you're using RHEL/CentOS 7.4 with the latest version of cloud-init-0.7.9, these steps might result in you losing connectivity to your instance after reboot. As a workaround, you can manually edit the /etc/sysconfigbetwork-scripts/ifcfg-eth0 file.To configure a RHEL/CentOS instance using cloud-initConnect to your instance using the instance's public IPv4 address.
