By default, AWS Identity and Access Management (IAM) users do not have permission to create or modify AWS RAM resources. To allow users to create or modify resources and perform tasks, you must create IAM policies that grant permission to use specific resources and API actions. You then attach those policies to the users or groups that require those permissions.Only the network owner can perform the following operations:Create a resource share.Create a core network.Update a resource share.View a resource share.View the resources shared by your account, across all resource shares.
