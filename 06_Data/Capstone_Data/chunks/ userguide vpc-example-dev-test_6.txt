Configure the subnets For Number of Availability Zones, choose 1. You can keep the default Availability Zone, or alternatively you can expand Customize AZs and select an Availability Zone. For Number of public subnets, choose 1. For Number of private subnets, choose 0. You can keep the default CIDR block for the public subnet, or alternatively you can expand Customize subnet CIDR blocks and enter a CIDR block. For more information, see Subnet CIDR blocks.For NAT gateways, keep the default value, None.For VPC endpoints, choose None. A gateway VPC endpoint for S3 is used only to access Amazon S3 from private subnets.For DNS options, keep both options selected. As a result, your instance will receive a public DNS hostname that corresponds to its public IP address.
