Under Policy version ID, choose the out-of-date policy version that you want to restore, and then choose Restore. The Policy version ID is incremented by one from the last version listed on the Policy versions page, and the Change set state displays as Pending generation. When generated, the Change set state changes to Ready to execute, and the Alias changes to LATEST. If any previous policies were in the Ready to execute change set state, those change to Out of date.
