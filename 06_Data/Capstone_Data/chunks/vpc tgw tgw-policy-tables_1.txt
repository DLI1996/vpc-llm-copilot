Transit gateway policy tables are currently only supported in Cloud WAN when creating a transit gateway peering connection. When creating a peering connection, you can associate that table with the connection. The association then populates the table automatically with the policy rules.For more information about peering connections in Cloud WAN, see Peerings in the AWS Cloud WAN User Guide.Create a transit gateway policy tableTo create a transit gateway policy table using the consoleOpen the Amazon VPC console at https://console.aws.amazon.com/vpc/.On the navigation pane, choose Transit gateway policy table.Choose Create transit gateway policy table.(Optional) For Name tag, enter a name for the transit gateway policy table. This creates a tag, where the tag value is the name that you specify.
